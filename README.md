# russian-cefr-aligner

## Описание проекта

**Участники проекта:** Анна Луценко, Мария Обридко, Анастасия Саунина

Наш проект заключается в обучении модели для упрощения текстов на русском языке до определённого уровня сложности CEFR с помощью алогритма обучения с подкреплением (RL).

Основные этапы работы:

1) Подготовка датасета текстов с отмеченным уровнем сложности по шкале CEFR;

2) Обучение модели-классификатора текстов по уровню сложности, дальнейшее использование обученной модели для написания reward-функции и оценки качества упрощения на тестовом датасете;

3) Написание reward-функции, которая учитывает качество упрощения (насколько упрощённый текст действительно близок к целевому уровню) и близость упрощённого текста к исходному (измеряется с помощью BERTScore и подобных метрик);

4) Сборка кода для применения rl-алгоритма с использованием библиотеки trl;

5) Проведение экспериментов с различными параметрами;

6) Итоговая оценка упрощения текстов.

Подробный отчёт о проекте можно найти по [ссылке](https://docs.google.com/document/d/1xJZlleAkNfEYnky8StCYoI60vOMrbWP2UTEdQpwYbCY/edit?usp=sharing).

## Структура репозитория

В папке ```data``` представлен код для получения датасета с текстами, классифицированными по уровню сложности, и сами файлы в формате .csv с train, test, val датасетами.

В папках ```CNN_classifier```, ```LLM_classifier```, ```feature_based_classifier```, ```finetuning``` находится код для дообучения различных типов классификаторов уровня сложности текстов. В ```finetuning``` представлен дообученный BERT, который используется в дальнейшем коде.

В файле ```test_models.ipynb``` происходит подбор LLM для симплификации текстов.

Наконец, в папка ```trl``` представлен финальный код, который собирает основные части: подбор LLM, определение функции вознаграждения, реализация обучения с алгоритмом rl, а также оценка качества симплификации на тестовом датасете. Также сравниваются две модели для упрощения: qwen и meno-tiny. Основные результаты представлены в readme-файле папки.
